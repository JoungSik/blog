# 오프라인 책장 프로젝트 설계

최근 만들고 싶은 프로젝트들이 따로 없어 심심함을 느끼고 있었습니다.

그러던 중 지인이 준 아이디어 중에 재밌어 보인다고 생각이 들었던 사이드 프로젝트를 설계하고 구현해볼까 합니다.

프로젝트 설계에 앞서 가장 중요한 이 프로젝트의 목적과 목표 입니다.

**오프라인에서 가지고 있는 책들을 온라인에서 확인 할 수 있는 서비스**

그리고 제 개인적인 목표로는 서버를 사용하지 않고 데이터베이스를 유저 스스로 관리 할 수 있는 서비스를 목표로 잡았습니다.

당장 떠올랐던 것은 노션에 데이터베이스를 구축하고 Notion API 를 사용해 데이터를 생성, 조회, 삭제, 수정 하는 방향을 생각 했습니다. 이전에 비슷한 GitHub Actions 를 만들었던게 생각이 나 구현에는 문제가 없다고 판단했습니다.

다음으로는 플랫폼입니다.

처음에는 PWA 웹앱과 웹 사이트도 고려를 했으나 RN을 통한 크로스 플랫폼으로 구현하겠습니다.

기능은 4가지 입니다.

- 책 검색하기
- 책 추가하기
- 책 수정하기
- 책 삭제하기

책 추가하기를 제외한 나머지 기능은 따로 설명이 필요 없기 때문에 설명을 생략합니다.

책 추가하기는 바코드를 인식시켜 해당 책의 이미지와 이름, 저자 같은 책의 정보를 가져와 DB에 저장하는 방식으로 구현합니다.

![1](/post/2021_12/2021_12_02/1.png)

아키텍처를 표현하자면 이렇게 됩니다.

책 정보 API 의 경우에는 지인이 아이디어를 줄 때 네이버 검색 했을 때 왠만하면 다 나온다는 이야기를 신뢰해서 [네이버 API](https://developers.naver.com/docs/search/book/) 를 쓰려고 합니다.

마지막으로 실제 배포를 진행하게 될 때는 노션에 데이터베이스 만드는법, 노션 API 등록하는 법, 등록된 데이터 베이스 ID 가져오는 법에 대한 가이드 라인을 앱 화면에서 어떻게 보여줘야 할지에 대한 대비까지가 프로젝트의 끝입니다.
